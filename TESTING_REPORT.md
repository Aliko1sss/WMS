# Стратегия тестирования ИС "Управление складом"

## 1. Виды тестирования
- **Unit:** проверка бизнес-логики (расчёты, валидация).
- **Integration:** проверка связки API ↔ БД ↔ модулей.
- **Smoke:** быстрая проверка основных эндпоинтов после сборки.

## 2. Критерии успешности
- ✅ Все тесты проходят в CI без ошибок
- ✅ Покрытие кода тестами критических участков (логика операций)
- ✅ 0 блокирующих багов в бета-версии
- ✅ Интеграционные тесты стабильны на чистой БД

## 3. Выявленные риски
| Риск | Влияние | Митигация |
|------|---------|-----------|
| Перерасход товара | Отрицательный остаток | Проверка остатка перед операцией "OUT" |
| Невалидный ввод | Ошибки 500 | Валидация через Pydantic |
| Параллельные запросы | Гонка данных | Использование транзакций БД |
